# Exercise 23 — PPX and Deriving

## Theory

### What Is a PPX?

**PPX** (PreProcessor eXtension) is OCaml's macro/metaprogramming system. A PPX is a program that transforms the OCaml AST before compilation. PPXs are invoked via dune's `(preprocess (pps ...))` stanza and are completely transparent to the runtime.

### ppx_deriving

`ppx_deriving` is a family of PPX plugins that automatically generate boilerplate code from type definitions using the `[@@deriving ...]` attribute.

```ocaml
type color = Red | Green | Blue
[@@deriving show, eq, ord]
```

This single annotation generates six functions:

| Plugin | Generated functions |
|--------|-------------------|
| `show` | `pp_color : Format.formatter -> color -> unit`<br>`show_color : color -> string` |
| `eq`   | `equal_color : color -> color -> bool` |
| `ord`  | `compare_color : color -> color -> int` |

### What Gets Generated

- **`show`** generates a `pp_*` printer using `Format` and a `show_*` wrapper calling `Format.asprintf`. For records it prints `{ field = value; ... }`, for variants it prints the constructor name.
- **`eq`** generates structural equality — equivalent to polymorphic `=` but type-safe and extensible via `[@equal ...]` attribute overrides.
- **`ord`** generates a comparison function compatible with `Map`, `Set`, and `List.sort`.

### Parametric Types

For `type 'a tree [@@deriving eq]`, the generated `equal_tree` takes an equality function for `'a` as its first argument:
```ocaml
val equal_tree : ('a -> 'a -> bool) -> 'a tree -> 'a tree -> bool
```

### Custom Printers

You can override the default printer with `[@printer fun fmt v -> ...]` on individual fields or constructors, giving you full control without losing the derived functions for other plugins.

## Practice Assignments

### Assignment 1: color_to_hex
Implement `color_to_hex` using a pattern match returning the appropriate hex string for each constructor.

### Assignment 2: color show and eq
Explore the functions generated by `[@@deriving show, eq, ord]`. Verify that `equal_color` and `compare_color` behave correctly for same and different constructors.

### Assignment 3: distance
Implement the Euclidean distance formula `sqrt((x2-x1)² + (y2-y1)²)` using the record fields of `point`.

### Assignment 4: tree_depth
Implement `tree_depth` recursively: `Leaf` has depth 0; a `Node` has depth `1 + max(left_depth, right_depth)`.

### Assignment 5: tree_insert
Implement BST insertion using Stdlib's polymorphic `compare`. Smaller values go left, larger go right, duplicates are ignored.
